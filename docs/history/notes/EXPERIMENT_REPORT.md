# PersonaPlex Companion Finetuning: Experiment Report

**Date:** April 3--5, 2026
**Model:** PersonaPlex (Moshi 7B variant)
**Objective:** Finetune a voice model on synthetic companion conversation data and evaluate whether finetuning improves companion persona quality over the base model with system prompting alone.

---

## 1. Background

### Model Architecture

PersonaPlex is a 7B-parameter voice model based on the Moshi architecture. It operates over three concurrent streams:

| Stream | Channels | Description |
|--------|----------|-------------|
| Text | 1 | Text token predictions |
| Agent audio | 8 | Audio codebook tokens (model's voice) |
| User audio | 8 | Audio codebook tokens (user's voice) |

This yields 17 total channels at a frame rate of 12.5 Hz with a 24 kHz sample rate.

### Training Setup

- **Method:** LoRA on the backbone transformer. Depformer and audio codec layers are frozen (`skip_depformer=True`).
- **Data:** 1,336 samples totaling 73.7 hours of synthetic companion conversation data, generated by Claude and voiced via VibeVoice TTS.
- **Batch size:** 16, with `duration_sec=180` (3-minute chunks).
- **Epoch length:** ~150 steps per epoch.

### Evaluation Protocol

Evaluation uses bot-to-bot dialogues across 5 companion scenarios, reviewed by Claude on three axes (each scored 1--5):

- **Coherence:** logical consistency and topical relevance
- **Naturalness:** conversational flow and human-likeness
- **Effectiveness:** success at the companion role (empathy, engagement, support)

---

## 2. Problem Statement

When finetuning on companion data, the model experiences **catastrophic forgetting**. Gen_eval scores start at 3--4 but collapse to 1--2 as training loss decreases. The primary symptom is the model generating excessive PAD (silence) tokens and filler words, producing incoherent or near-silent outputs.

---

## 3. Phase 1: Diagnosing the PAD Collapse

### Root Cause Discovery

Analysis of generated transcripts revealed PAD token percentage increasing monotonically with training steps, even as text eval loss decreased:

| Step | PAD % | Real text % | Coherence | Text eval loss |
|------|-------|-------------|-----------|----------------|
| 32 | 74.0% | 16.7% | 3.6 | 0.789 |
| 64 | 88.1% | 8.5% | 1.8 | 0.714 |
| 96 | 96.5% | 3.0% | 1.2 | 0.681 |

**Root cause:** In 210-second training chunks at 12.5 Hz, only ~10% of text frames contain actual words. The remaining ~90% are padding tokens (`text_padding`, `end_of_text_padding`). Even with `text_padding_weight=0.1`, padding drove approximately 27% of the effective gradient. The model learned to predict PAD everywhere as the path of least resistance --- a degenerate optimum that minimizes loss while producing silence.

### Fix: `text_padding_weight = 0.0`

Setting the padding weight to zero fully masks padding tokens from the text loss gradient. The existing loss computation code already handled this case correctly via a `weights > 0.0` guard. This single change eliminated PAD collapse in all subsequent runs.

### Diagnostics Added

To monitor this failure mode going forward, the following metrics were added to both training and evaluation logging:

- **`real_token_loss`**: Cross-entropy loss computed on actual speech tokens only
- **`pad_token_loss`**: Cross-entropy loss computed on padding tokens only
- **`real_token_pct`**: Percentage of text frames containing real (non-padding) tokens
- **`pred_pad_pct`**: Percentage of the model's greedy predictions that are padding tokens

## 4. Phase 2: L2 Regularization

### Implementation

An L2 penalty on LoRA parameters was added to prevent excessive drift from the base model weights.

### Bug: Penalizing `lora_A` vs `lora_B`

The first implementation penalized all LoRA parameters. This was incorrect due to LoRA's initialization scheme:

- `lora_A`: initialized with Kaiming random values (non-zero)
- `lora_B`: initialized to zeros
- Output: `lora_B @ lora_A * scaling`

Only `lora_B` growing from zero represents actual drift from the base model. Penalizing `lora_A` wasted gradient budget fighting a random initialization, manifesting as a constant ~0.289 L2 loss component that never changed regardless of training progress.

**Confirmation:** Run v0-15 (broken L2 on all params) performed identically to v0-14 (no L2), confirming the penalty had no functional effect. After fixing to target `lora_B` only, L2 regularization worked as intended.

## 5. Phase 3: Hyperparameter Search

### Pre-Padding-Fix Runs (all collapsed)

These runs used `text_padding_weight=0.1` and all suffered PAD collapse:

| Run | Key params | Step 96 avg score |
|-----|-----------|-------------------|
| v0-10 | pad_w=0.1, lr=5e-6, rank=64 | 1.4 |
| v0-13 | pad_w=0.1, lr=5e-6, rank=32 | 1.3 |

### Post-Padding-Fix Runs

All runs below use `text_padding_weight=0.0` and `audio_loss_weight=0.5` unless noted.

| Run | LR | Rank | L2 | Batch | Notes | Best avg | At step | Trajectory |
|-----|-----|------|----|-------|-------|----------|---------|------------|

| v0-15 | 2e-6 | 64 | 1e-4 (broken) | 16 | L2 was no-op | 2.7 | 96 | |

| v0-17 | 1e-6 | 32 | 1e-3 (fixed) | 16 | | 3.2/3.3 | 96/224 | Sustained ~2.5--2.9 post-peak |

| v0-23 | 4e-6 | 64 | 1e-3 | 16 | | 2.8 | 16 | Too aggressive, declined |
| v0-24 | 2e-6 | 64 | 1e-3 | 16 | microbatch=2 (eff batch 32) | 3.0 | 16 | Killed for overnight run |
| v0-25 | 5e-6 | 64 | 1e-3 | 16 | | 2.3 | 32 | Killed, too fast |
| v0-26 | 2e-6 | 128 | 1e-3 | 16 | | 3.1 | 32 | More capacity = faster overfit |
| v0-27 | 5e-6 | 128 | 1e-3 | 16 | | 2.6 | 32 | Killed |
| v0-28 | 1e-5 | 64 | 5e-3 | 16 | | 1.0 | 48 | Collapsed despite strong L2 |
| v0-29 | 2e-6 | 64 | 1e-3 | 16 | text-only (audio_loss=0.0) | 3.1 | 32 | Same quality as with audio |

### Observations

- **Higher LoRA rank accelerates memorization, not generalization.** Rank 128 peaks sooner and overfits faster than rank 64 or 32.
- **Higher learning rates consistently hurt.** LR >= 4e-6 leads to rapid quality decline. Pairing aggressive LR with strong L2 (v0-28: lr=1e-5, L2=5e-3) does not compensate --- the model still collapses.
- **Text-only training (v0-29, audio_loss=0.0)** produces the same quality as joint text+audio training while being 2x faster. Audio loss appears neutral for companion quality.
- **Best identified config:** lr=2e-6, rank=64, L2=1e-3, audio_loss=0.5, batch=16, duration=180s (v0-22).

## 7. Phase 5: Sampling Parameter Discovery

Six sampling configurations were tested on both the base and finetuned models to assess whether inference-time parameters could yield larger gains than training changes.

### Base Model

|--------|-----------|-----------|------------|------------|-----------|-------------|---------------|-----|

| baseline | 0.4 | 15 | 0.45 | 80 | 3.0 | 3.2 | 3.2 | 3.1 |

### Finetuned Model

| Config | temp_text | topk_text | temp_audio | topk_audio | Coherence | Naturalness | Effectiveness | Avg |
|--------|-----------|-----------|------------|------------|-----------|-------------|---------------|-----|
| cold | 0.2 | 5 | 0.3 | 40 | 3.8 | 3.4 | 2.6 | 3.3 |

### Key Finding

The `warm_balanced` configuration (`temp_text=0.7`, `topk_text=30`, `temp_audio=0.55`, `topk_audio=100`) produced a **+0.4 average improvement** over the baseline sampling on the base model --- a larger single-factor gain than any training hyperparameter change achieved. Cold sampling preserves coherence but sacrifices naturalness. Hot sampling degrades everything. The sweet spot is moderate warmth with a constrained but not overly narrow top-k.

To validate the evaluation pipeline and establish an upper bound, ideal text-only conversations were generated by Claude and run through the same LLM review pipeline.

| Metric | Score |
|--------|-------|
| Coherence | 5.0 |
| Naturalness | 4.0 |

This confirms the evaluation pipeline is functioning correctly and that models have significant headroom. The naturalness ceiling of 4.0 (rather than 5.0) likely reflects the inherent artificiality of text-only conversations lacking prosodic and timing cues.

## 9. Phase 7: Repetition Penalty

A token-level repetition penalty was implemented in the sampling code. It penalizes the logits of recently generated tokens by a multiplicative factor.

### Results (Base Model)

| Repetition penalty | Coherence | Naturalness | Effectiveness | Avg |
|-------------------|-----------|-------------|---------------|-----|
| 0 (none) | 3.0 | 3.2 | 3.0 | 3.1 |
| 1.2 | 3.0 | 3.2 | 3.4 | 3.2 |
| 1.3 | 2.8 | 2.8 | 3.0 | 2.9 |
| 1.5 | 2.6 | 2.6 | 2.6 | 2.6 |
| 2.0 | 2.6 | 2.4 | 3.0 | 2.7 |

A marginal gain appears at penalty=1.2, but higher values actively degrade quality. The model's repetition problem operates at the phrase and structure level ("I think it's, I think it's"), not at the individual token level. Token-level penalties cannot address this and instead disrupt normal language patterns.

## 10. Qualitative Failure Analysis

Review of generated transcripts across multiple runs revealed consistent failure modes:

1. **Repetitive loops:** The model frequently enters phrase-level repetition cycles ("I think it's, I think it's", "like, like, like"), particularly on emotional or open-ended topics.

2. **Fabrication:** The model invents personal experiences ("I'm in Portland", "I played board games"), fabricates TV show titles, and states false facts with confidence. This likely reflects patterns absorbed from the synthetic training data.

3. **Hedging on emotional topics:** When faced with genuine emotional content, the model resorts to excessive filler ("um", "you know") and generic advice rather than active listening or empathetic engagement.

4. **Casual/open-ended topics are hardest:** Without clear conversational structure, the model fills the void by inventing content or cycling through surface-level observations.

5. **Structured emotional topics work best:** Scenarios with inherent direction (relationship advice, life decisions) produce the most coherent and useful outputs, as they provide a natural conversational scaffold.

## 11. Code Changes

The following modifications were made to the codebase during this experiment session:

### Training Infrastructure
1. **`text_padding_weight=0.0` support** (`loss.py`) --- Mask padding tokens from text loss gradient.
2. **Decomposed text loss diagnostics** (`loss.py`, `train.py`, `eval.py`) --- Added `real_token_loss`, `pad_token_loss`, `real_token_pct`, `pred_pad_pct` metrics.
3. **L2 regularization on `lora_B` only** (`train.py`, `args.py`) --- Penalize drift from base model without interfering with `lora_A` initialization.
4. **`force_warm_restart` flag** (`train.py`, `args.py`) --- Enable resuming training with a fresh learning rate scheduler.

### Inference and Evaluation
5. **Updated sampling defaults to `warm_balanced`** (`args.py`) --- `temp_text=0.7`, `topk_text=30`.
6. **Repetition penalty in generation** (`lm.py`, `bot_to_bot.py`) --- Token-level frequency penalty on recently generated tokens.
7. **`test_sampling_params.py`** --- Sampling parameter sweep tool.
8. **`run_multi_seed_eval.py`** --- Multi-seed statistical evaluation tool.
9. **`eval_text_ceiling.py`** --- Text-only conversation ceiling test.

## 12. Summary of Findings

1. **`text_padding_weight=0.0` is essential.** Without it, the model inevitably collapses into predicting PAD tokens everywhere, regardless of other hyperparameters. This is the single most important training configuration choice.

2. **Optimal training configuration:** lr=2e-6, rank=64, L2=1e-3 (on `lora_B` only), audio_loss=0.5, batch=16, duration=180s.

3. **Sampling parameters matter more than training.** The `warm_balanced` configuration (temp_text=0.7, topk_text=30) produced the largest single-factor quality improvement (+0.4 avg), exceeding the effect of any training hyperparameter change.

4. **Finetuning is statistically neutral for companion persona.** Multi-seed evaluation (n=50) shows no meaningful difference between finetuned and base models. The base model with a companion system prompt already captures the target behavior.

5. **The base model + system prompt is sufficient.** Finetuning on synthetic companion data does not teach the model new capabilities it did not already possess.

6. **Remaining quality gap is substantial (3.3 vs 5.0 ceiling)** and is dominated by three factors: repetitive generation (architectural limitation), fabrication (data artifact), and hedging on emotional topics (data artifact).

7. **Higher LoRA rank accelerates memorization, not generalization.** Rank 128 peaks sooner and overfits faster than rank 64.

8. **Higher learning rates consistently hurt.** Aggressive LR paired with strong L2 does not compensate --- the model still collapses.

9. **Audio loss is neutral.** Text-only training (audio_loss=0.0) produces equivalent quality at 2x the training speed.

---

## 13. Recommended Next Steps

1. **Enable puppeteer/context injection training.** This represents a genuinely new capability the base model lacks and is the most promising direction for finetuning to add value.

2. **Improve training data quality.** Remove fabrication patterns from the synthetic data. Improve emotional scenario handling with more nuanced, listening-oriented responses rather than generic advice.

3. **Consider DPO instead of SFT.** Preference learning may generalize better than directly imitating synthetic data, particularly for subtle conversational qualities like empathy and active listening.

4. **Adopt `warm_balanced` sampling as default.** Use `temp_text=0.7`, `topk_text=30`, `temp_audio=0.55`, `topk_audio=100` for all evaluation and production serving.
