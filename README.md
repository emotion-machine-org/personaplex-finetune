# PersonaPlex Insurance Broker Voice Training

> **Status: WIP. Not tested on a wide range of setups - we welcome any and all contributions for improving developer experience.**

Finetune [PersonaPlex](https://huggingface.co/nvidia/personaplex-7b-v1) (a Moshi 7B variant with voice + role conditioning) on a specific domain like pharmaceutical adherence for use in real-time voice conversations, with mid-conversation grounding via a puppeteer LLM that injects context into the model's text stream.

## What we did with this code

We used this pipeline through March–April 2026 to finetune PersonaPlex on two domains: **insurance brokerage** (~3,800 synthetic dialogues, ~389 hours of audio) and **pharmaceutical patient support** (~2,000 calls, ~166 hours), all generated by Claude and voiced with VibeVoice. Across roughly 84 training runs we hit and worked through five recurring failure modes — depformer LoRA corruption, train/inference channel mismatches during context injection, PAD-token collapse, badly applied L2 regularization, and injection dropout at chunk boundaries. The two non-obvious fixes that mattered most were `text_padding_weight=0.0` (which eliminated the PAD-collapse degenerate optimum) and L2 regularization on `lora_B` weights (which prevented catastrophic forgetting at the low learning rates we needed for stable generation). Our best pharma checkpoint reached a **4.84/5 grounding score** with the puppeteer context-injection mechanism. A surprising null result on the companion side: pure finetuning was statistically neutral against the base model with a system prompt — sampling parameters mattered more than any training change. Full writeup, configs, and unsampled metric history for all 59 logged runs are in [`docs/history/`](docs/history/).

## Overview

The pipeline generates synthetic broker/client dialogues with embedded context injections, renders them to audio, and finetunes PersonaPlex to (a) speak grounded in provided facts, (b) stay silent while receiving injected context, and (c) hedge when information isn't available.

```
                                              [1] generate_dialogues_sync.py
                                                  Claude API, 8-dim LHS
                                                        │
                                                        ▼
outbound_insurance.jsonl ─────────── text_prompt + dialogue + context_injections
                                                        │
                            ┌───────────────────────────┤
                            ▼                           ▼
               [2] parse_dialogues.py         (injection metadata preserved
                    scripts/{id}.txt           for later frame offset computation)
                            │
                            ▼
               [3] generate_audio.py          VibeVoice 7B TTS (multi-GPU)
                    mono_wav/{id}.wav
                            │
                            ▼
               [4] create_stereo.py           WhisperX align + channel routing
                    --dialogues *.jsonl       propagates text_prompt + context_injections
                    stereo_wav/{id}.wav+.json  both-speaker alignments + turns metadata
                            │
                            ▼
               [5] compute_injection_offsets.py   after_turn → frame_offset
                    stereo_wav/{id}.json          (proactive/reactive placement)
                            │
                            ▼
               [6] create_manifest.py
                    dataset/{train,eval}.jsonl
                            │
                            ▼
               [7] moshi-finetune train.py    FSDP + LoRA (multi-GPU)
                    runs/                     context injection splicing + loss masking
```

At inference, an external puppeteer LLM injects `<context>` tokens into the text stream via a hidden channel. The model reads the context through the transformer but the tokens are suppressed from all external output (websocket, transcripts, silence metrics).

> **Related work.** Sakana AI's [KAME (Tandem Architecture for Enhancing Knowledge in Real-Time S2S Conversational AI)](https://pub.sakana.ai/kame/) ([arXiv:2510.02327](https://arxiv.org/abs/2510.02327), accepted at ICASSP 2026, posted April 30, 2026) describes the same core idea — a backend LLM asynchronously enriching a fast speech-to-speech model with knowledge during the user's turn — under a different name and with a different architecture (parallel "tandem" model rather than in-stream text-token splicing). Our puppeteer work was developed independently in March–April 2026 ahead of this release; we recommend KAME as the more thoroughly evaluated reference for this class of approach.

## Pipeline steps

### 1. Generate dialogues

Generate synthetic broker/client dialogues with structured system prompts and context injection markers using Claude.

```bash
.venv/bin/python pipeline/generate_dialogues_sync.py \
  --output outbound_insurance.jsonl \
  --num-dialogues 500 \
  --workers 10
```

Each record contains:

- `text_prompt`: Structured pre-call brief (ROLE / CLIENT / KNOWN / UNKNOWN / GOAL / STYLE)
- `dialogue`: Full transcript with `BROKER (Name):` / `CLIENT (Name):` turns
- `context_injections`: `[{after_turn: int, text: str}, ...]` — directive coaching instructions

**8-dimensional Latin Hypercube Sampling** ensures diversity across broker persona, client persona, outbound goal, emotional tenor, domain vocabulary, complexity, call outcome (80/20 success/failure), and injection count (0/3/6/9 weighted).

75% of dialogues include context injections (directive coaching mid-conversation), 25% without (teaches hedging on unknowns).

### 2. Parse dialogues

```bash
.venv/bin/python pipeline/parse_dialogues.py data/companion_training/companion_training.jsonl data/companion_training/scripts 6000
```

Converts JSONL transcripts to VibeVoice `Speaker N:` format. Strips `[Call start/end]` markers, normalizes unicode, skips dialogues over 3000 words.

### 3. Generate mono audio

```bash
.venv/bin/python pipeline/generate_audio.py \
  --model-path vibevoice/VibeVoice-7B \
  --dialogues data/companion_training/companion_training.jsonl \
  --output-dir data/companion_training/mono_wav \
  --scripts-dir data/companion_training/scripts \
  --voices-dir data/companion_training/speaker_samples \
  --gpus 0 --resume --batch-size 16
```

VibeVoice 7B TTS with deterministic per-dialogue voice assignments. `--gpus` spawns one process per GPU.

### 4. Create stereo WAVs + alignments

```bash
.venv/bin/python pipeline/create_stereo.py \
  --mono-dir data/<dataset>/mono_wav \
  --scripts-dir data/<dataset>/scripts \
  --output-dir data/<dataset>/stereo_wav \
  --dialogues data/<dataset>/<dataset>.jsonl \
  --whisper-model large-v3 \
  --gpus 0 --resume --batch-size 16
```

1. WhisperX transcribes mono audio → word-level timestamps via wav2vec2 forced alignment
2. Greedy sequential matching maps words to ground-truth speaker labels
3. Broker → left channel, client → right channel (10ms crossfade)
4. Writes both-speaker alignments (`SPEAKER_BROKER`/`SPEAKER_CLIENT`), explicit `turns` metadata, and propagates `text_prompt` + `context_injections` from the JSONL

### 5. Compute injection frame offsets

```bash
.venv/bin/python pipeline/compute_injection_offsets.py --stereo-dir data/<dataset>/stereo_wav
```

Converts `after_turn` (turn index) to `frame_offset` (absolute frame at 12.5 Hz) using preserved turn metadata:

- **Proactive (~70%)**: Turn before marker is CLIENT → `frame_offset = start of client utterance` (context loads during client speech, zero perceived latency)
- **Reactive (~30%)**: Turn before marker is BROKER hedge → `frame_offset = end of broker utterance` (context loads during pause after "let me check on that")

### 6. Create manifest

```bash
.venv/bin/python pipeline/create_manifest.py \
  --stereo-dir data/<dataset>/stereo_wav \
  --train-output data/<dataset>/dataset/train.jsonl \
  --eval-output data/<dataset>/dataset/eval.jsonl \
  --eval-fraction 0.05
```

### 7. Finetune

```bash
cd moshi-finetune
CUDA_VISIBLE_DEVICES=0 .venv/bin/torchrun --nproc-per-node 1 \
  -m train ../configs/pharma_demo.yaml
```

| Parameter | Value | Notes |
|-----------|-------|-------|
| Base model | `nvidia/personaplex-7b-v1` | PersonaPlex 7B (dep_q=16) |
| LoRA rank | 128 | ~5.6% params trainable (~500M / 8.9B), depformer excluded (`skip_depformer`) |
| duration_sec | 60 | Sequence length per sample |
| batch_size | 4 | Per-GPU; effective = batch_size x num_gpus |
| max_steps | 3000 | |
| lr | 2e-6 | OneCycleLR with 5% warmup |
| system_prompt | enabled | Voice + text prompt prefix, loss masked |
| puppeteer | enabled | Context injections spliced into text channel |

**Context injection during training:** The interleaver reads `context_injections` from each sample's JSON, wraps text in `<context>...</context>` tags, and inserts frames into the sequence at `frame_offset`. Agent audio is set to `SILENCE_TOKENS` (broker stays quiet); user audio is preserved from the displaced conversation frames (client may be speaking — ~70% proactive injections). A `context_mask` is built over injection frames: text loss is zeroed (model didn't produce these tokens); audio loss stays active (model learns silence for agent audio while tracking user audio).

**Context injection at inference:** `lm.py` forces agent audio output to `SILENCE_TOKENS` when `_injecting_context` is True, matching the training signal. User audio input flows through unmodified (preserved by the `provided` mask in the cache update).

### 8. Merge LoRA + serve

```bash
# Merge
.venv/bin/python pipeline/merge_lora.py \
  --checkpoint runs/<run_name>/checkpoints/checkpoint_000128/consolidated \
  --output runs/<run_name>/merged/model.safetensors

# Serve
cd personaplex/moshi
.venv/bin/python -m moshi.server \
  --moshi-weight ../../runs/<run_name>/merged/model.safetensors \
  --static ../../personaplex/client/dist
```

The server accepts mid-session context injection via JSON websocket messages:

```json
{"type": "context", "text": "Quote the Berkshire at $13,100. Emphasize savings."}
```

## Evaluation

### Gen eval (during training)

Runs Gemini Live ↔ Moshi dialogues at checkpoint time, evaluates coherence/naturalness/effectiveness/grounding via Claude, logs to W&B.

```bash
python -m finetune.gen_eval \
  --checkpoint runs/.../consolidated \
  --run-dir runs/... --step 200 \
  --config configs/pharma_demo.yaml
```

### Batch eval (standalone)

30 dialogues (10 seeds x 3 scenarios), two in parallel across GPU pairs.

```bash
python pipeline/batch_eval.py \
  --checkpoint runs/.../consolidated \
  --output-dir runs/.../batch_eval \
  --gpus 1,2,3,4

# With puppeteer (scheduled context injections):
python pipeline/batch_eval.py \
  --checkpoint runs/.../consolidated \
  --output-dir runs/.../batch_eval_puppeteered \
  --gpus 1,2,3,4 \
  --puppeteer knowledge_base.json
```

Puppeteer knowledge base format: `{scenario_id: [{frame: int, text: str}, ...]}`.

### Compare runs

Statistical comparison with Wilcoxon tests, Cliff's delta, bootstrap CIs, control charts, and automated grounding rate (factual claim extraction vs provenance).

```bash
python pipeline/compare_runs.py \
  --run-a runs/base_eval/results.json \
  --run-b runs/lora_eval/results.json \
  --output runs/comparison_report
```

Produces `comparison.json` + `dashboard.html` with interactive Chart.js visualizations.

## Architecture

- **Moshi 3-stream model:** agent text (1ch) + agent audio (8 codebooks) + user audio (8 codebooks) = 17 channels
- **PersonaPlex dep_q=16** (vs standard Moshi dep_q=8) — depformer handles all 16 audio codebooks
- **Mimi codec:** 12.5 Hz frame rate, 24kHz sample rate, 8 codebooks per audio stream
- **System prompt prefix:** voice prompt → silence → `<system>` text prompt `</system>` → silence → conversation
- **Context injection:** `<context>` tokens spliced into text channel mid-conversation; hidden from all external output via `_injecting_context` flag. Agent audio forced to silence during injection (both training and inference); user audio preserved from conversation (client may be speaking)
- **Loss masking:** Prompt region masked via `prompt_lengths`; injection region masked for text loss only via `context_masks` (audio loss stays active — agent learns silence, user audio predicted normally)
- **LoRA scope:** Applied to backbone transformer only (`skip_depformer=True`); depformer is frozen to preserve pretrained audio codec generation
- **Queue priority:** context > nudge > greeting (context injection takes precedence over silence-breaking nudges)
